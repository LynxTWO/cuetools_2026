## 2026-07-02 — Measurement foundation complete

- Pristine LAME 3.100 committed to `c:\DEV\lame_v4` (own git history); `build.cmd` = one-command 64-bit build.
- **Bit-exact regression** (`tests/regress.ps1`) over a deterministic synthetic corpus (`tests/regress_corpus`, regenerated by `tests/corpusgen`): 70-case baseline of MP3 + decoded-PCM SHA-256. Green on pristine build. This is the behavior-preserving gate.
- **Encoder-independent perceptual meter** (`tests/nmr`): derives its own masking model (critical-band + spreading + ATH), scores decoded-vs-original; primary `meanNMRdb` (lower = more transparent). Validated monotone with bitrate (music: 128k −3.5 → 320k −15.3 dB) and self-transparent (file vs itself ≈ −100 dB).
- **A/B quality harness** (`tests/abtest.ps1`): corpus × two builds → per-file + mean `meanNMRdb` delta. delta<0 = variant more transparent at equal bitrate. A=B self-check 0.000.
- Corpus split: synthetic set for the bit-exact gate; `tests/corpus/` holds the synthetic set **plus user-supplied real killer samples** (incl. Tom's Diner a cappella) for quality eval. All corpus WAVs git-ignored (regenerable and/or copyrighted).

## 2026-07-02 — Q-A experiment 1: in-loop Huffman search (best_huffman=2)

**Hypothesis:** stock `-q0` disables the thorough in-loop Huffman optimization (`use_best_huffman=2`, "best (inside loop, slow)") purely for speed (its own comment says so). Costing every candidate quantization with optimal Huffman partitioning should let the outer loop afford finer quantization at a fixed bit budget → lower noise.

**Method:** branch `qa/deep-search`, set q0 → `use_best_huffman=2`, rebuild, `abtest.ps1` A=stock vs B=variant at `-q0 -b128` on the synthetic corpus.

**Result (measured):** corpus-mean `meanNMRdb` delta **−0.030 dB** (B more transparent). 6/10 files better (best: `hf_square` −0.21 dB, `music_mix` −0.08), 2 marginally worse (≤0.05), 2 neutral. **Small but real, and the harness detected it end-to-end** — proving the measurement loop works on actual encoder changes.

**Design finding:** the regression gate flagged drift in 12 cases, not the expected 10 — `-V0`/`-V5` VBR also drifted on some content, because LAME's VBR presets reuse the quality=0 config internally. **Therefore the production `--quality max` must be a dedicated flag/mode, not a change to `-q0`,** or it would silently alter `-V0`/`-V5` output (forbidden without ABX). The experiment branch keeps this change isolated; master stays pristine.

**Verdict:** best_huffman=2 is a legitimate (small) win to fold into `--quality max`. The large gains require the real flagship — a genuinely deeper noise-allocation search (trellis/exhaustive scalefactor + global-gain), next.
